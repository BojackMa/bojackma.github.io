1. U-Net 是什么，从哪里来

U-Net 诞生于 2015 年，由弗莱堡大学的 Ronneberger、Fischer 和 Brox 提出，最初是专门为医学图像分割任务设计的神经网络架构。

它的名字非常直白——把网络结构画出来，长得像字母 U。

在当时，医学图像标注数据极其稀少（给每张细胞照片精确标注需要专业医生，成本极高），U-Net 的一个重要贡献是能在极少量训练数据下取得很好的效果。

后来，U-Net 凭借其优雅的结构被图像生成领域大量借用，成为早期扩散模型（Stable Diffusion 1.x、2.x、SDXL）的核心去噪网络，在 AI 绘图的黄金时代扮演了至关重要的角色。

2. 理解 U-Net 之前，先理解 CNN 怎么”看”图像

卷积核：局部扫描

CNN 用一个小窗口（卷积核）在图像上滑动，提取局部特征。每个卷积核学会识别某种模式：有的识别横向边缘，有的识别纵向线条，有的识别颜色变化。

特征图（Feature Map）与通道数

每个卷积核扫描整张图后，输出一张特征图。用 64 个卷积核，就得到 64 张特征图，叠在一起就是：

这里的 64 就是通道数，代表”从 64 个不同角度观察这张图得到的信息”。

比喻：就像用 64 种不同的滤镜分别拍同一张照片，每张滤镜照片强调不同的特征，叠在一起就是 64 通道的特征图。

下采样：越深越抽象

随着层数加深，通过池化操作（Max Pooling）对特征图进行空间压缩：

空间尺寸越小（视野越大，感受野越广），通道数越多（需要更多维度来表达复杂语义）。这是一个”以空间换语义”的过程。

3. 图像分割：U-Net 要解决的原始问题

任务定义

给一张细胞显微镜照片，对每个像素打标签：属于细胞（1）还是背景（0）。

输出必须和输入完全一样大，精确到像素级别。

核心矛盾

这个任务天然存在两个互相矛盾的需求：

需求 为什么需要 代价
大视野（语义理解） 判断一个像素是否属于细胞，需要看整体上下文 必须压缩图像，丢失空间细节
精细定位（像素级精度） 细胞边界必须精确到像素 必须保留原始空间信息

压缩图像会丢失精细信息，这是 CNN 的内在矛盾。

普通的编码器-解码器结构（Encoder-Decoder）虽然能还原尺寸，但压缩过程中丢失的边缘细节无法找回，导致分割边界模糊。

4. U-Net 的结构：为什么是 U 形？

U-Net 的解决方案是：用两条并行的路径分别处理两类信息，最后合并。

这个结构从侧面看：左边向下、右边向上、中间有横向连接，正好构成一个 U 字形。

横向的箭头，就是跳跃连接（Skip Connections）——这是 U-Net 最核心的创新。

5. 三个核心问题详解

问题一：通道数（如 64）是什么？

通道数 = 从多少个角度观察图像。

  • 原图的 3 个通道 = RGB 三种颜色
  • 卷积后的 64 个通道 = 64 种不同的特征视角（边缘、纹理、颜色梯度……）
  • 通道数越多，表达能力越强
  • 越深的层通道数越大（64 → 128 → 256 → 512），因为深层概念更复杂，需要更多维度来表达

问题二：为什么解码器变成了 128 通道？

以第一层为例：

  • 编码器第一层输出：512×512×64
  • 解码器上采样输出：512×512×64
  • 两者拼接之后：512×512×128

多出来的 64 通道，来自编码器通过跳跃连接传来的特征图。

问题三：什么是拼接（Concatenation）？

拼接是在通道维度上把两个特征图合并，空间尺寸不变，通道数相加。

用积木来理解：把两叠 64 张纸摞在一起，变成一叠 128 张纸。每张纸的大小（512×512）没变，只是纸的数量翻倍了。

拼接之后，后续卷积层能同时”看到”：

  • A 的信息：编码器保留的空间细节（边缘、轮廓的精确位置）
  • B 的信息：解码器路径的语义信息（这里是细胞、那里是背景）

6. 跳跃连接的真正作用

信息丢失问题

编码器压缩图像时，像素级别的位置信息不断丢失：

到了瓶颈层（64×64），模型知道”左上角有个细胞核”，但已经不知道细胞轮廓精确在哪些像素上了——这个信息在下采样时丢掉了。

解码器试图还原，但”找不回”的细节只能靠猜，导致边界模糊。

跳跃连接的解决方式

在编码器压缩之前，把那一层的特征图”拍一张快照”存起来，等解码器还原到同样尺寸时，直接把快照拿过来用。

这些快照里保留的，正是下采样之前的精细空间信息。

两条信息流的分工

信息流 来源 包含什么
语义流 瓶颈层 → 解码器（从下往上） 画什么、在哪个区域
细节流 编码器 → 跳跃连接（横向） 边缘在哪、轮廓怎么走

两条流在解码器每一层合并，模型同时拥有语义理解和空间细节：

比喻：就像你临摹一幅画，不仅记住了整体构图（语义），还在每个阶段拍了参考照片（细节流）。画到哪个精细程度，就拿出对应阶段的照片来参考。

7. U-Net 如何被扩散模型借用

扩散模型在做什么？

扩散模型生成图像的过程：

每一步需要一个网络回答：给定当前这张有噪声的图，应该去掉哪些噪声？

  • 输入：当前有噪声的图（比如 64×64×4 的潜空间特征）
  • 输出：预测的噪声（和输入完全一样大）

为什么 U-Net 天然适合？

看到任务形式，你会发现：

医学图像分割 扩散模型去噪
输入 有细胞的图像 有噪声的图像
输出 每个像素的标签 每个位置的噪声预测
尺寸要求 输入 = 输出 输入 = 输出

任务形式几乎完全一样：输入一张图，输出一张同样大小的图。

U-Net 天生满足这个约束，直接拿来用，只需要加两个条件注入机制。

加入时间步条件

扩散模型需要知道”现在是第几步去噪”——第1步噪声很多要大幅去噪，第49步噪声很少要精细调整。

做法：把时间步 t 编码成一个向量，注入 U-Net 每一层（通过加法或自适应层归一化）。

加入文本条件

用户输入的文字描述（”一只红色的猫”）需要引导去噪方向。

做法：文本先被 CLIP 或 T5 编码成向量序列，在 U-Net 的中间层通过 Cross-Attention 注入：

图像的每个位置去”询问”文本，哪些文字描述和这个位置最相关？由此让图像特征朝着文本描述的方向调整。

为什么去噪也需要跳跃连接？

去噪任务同样需要两类信息：

  • 全局语义（编码器路径）：这张图整体在画什么，大结构应该怎样
  • 局部细节（跳跃连接）：这个区域的纹理、边缘应该怎样精细还原

没有跳跃连接，高频细节（发丝、文字笔画、皮肤纹理）在去噪过程中很难准确还原，生成结果会糊。

Stable Diffusion 的完整流程

Stable Diffusion 在 U-Net 基础上还引入了潜空间（Latent Space）：

U-Net 不在原始像素上工作，而是在压缩了 8 倍的潜空间上工作，计算量大幅降低，这就是”Latent Diffusion Model（LDM）”的核心思路。

8. U-Net 的局限性与被取代

U-Net 统治图像生成领域近三年，但它有两个先天缺陷：

缺陷一：感受野受限

卷积操作只能看到局部区域。建立远程依赖（让图像左上角的人物和右下角的背景风格一致）需要很多层才能”传递”信息，效率低。

Transformer 的 Self-Attention 天然全局——每个位置可以直接和任意其他位置交互，一步到位。

缺陷二：Scaling 不规律

把 U-Net 做大，效果提升不稳定、不可预测。Transformer 有明确的 Scaling Law（模型越大效果越好，且规律可预测），方便大厂堆资源提升质量。

继任者：DiT（Diffusion Transformer）

2022 年，DiT 把去噪网络从 U-Net 换成纯 Transformer：把潜空间特征图切成 patch，当成 token 序列，用标准 Transformer blocks 处理。

从 Stable Diffusion 3 开始，U-Net 被 DiT 取代，此后 FLUX、WAN、Seedream、Sora、Veo 全部采用 DiT 架构。U-Net 在图像生成领域基本退出了历史舞台。

9. 为什么是 Transformer，而不是改进 U-Net？

U-Net 的两个缺陷已经明确——感受野局限和 Scaling 不规律。但你可能会问：为什么不直接修补 U-Net，而是整体换掉？

根本原因：两种架构的设计哲学不同

U-Net 的核心是卷积，卷积天生是局部操作：一个 3×3 的卷积核只能看到 3×3 的窗口。要让模型”看到”全局，只能靠不断堆叠层数，信息一层一层向上传递。

这本质上是一种间接的全局视野——通过堆积局部操作来模拟全局理解，效率先天不足。

Transformer 的 Self-Attention 则完全不同：

这是直接的全局视野，一层 Attention 就完成了全局建模。

修补 U-Net 的尝试为什么不够？

研究者确实尝试过在 U-Net 中加入 Attention 模块（Stable Diffusion 1.x/2.x 的 U-Net 里就有 Self-Attention 层）。但这种方案是在卷积骨架上打补丁，带来了新问题：

问题 原因
架构复杂，维护成本高 卷积层 + Attention 层混合，超参数繁多
Scaling 行为不一致 卷积部分和 Attention 部分对参数量的响应不同，难以统一放大
编码器-解码器不对称 U 形结构本身的设计是为分割任务定制的，在生成任务中是历史包袱

DiT 的做法：彻底抛弃 U 形结构

DiT（Diffusion Transformer，2022）的核心思路是激进的：

完全不用卷积，不用编码器-解码器，不用跳跃连接。把图像当成序列，用纯 Transformer 处理。

具体做法：

图像的每个 patch 变成一个 token，和 NLP 里的词向量地位完全相同。时间步 t 和文本条件通过 AdaLN（自适应层归一化）注入，不再需要专门的条件注入设计。

这个架构极其简洁：没有下采样、没有上采样、没有跳跃连接，整个网络是等深等宽的同质结构（Isotropic Architecture）。

同质结构为什么对 Scaling 友好？

U-Net 的 U 形结构意味着不同层的功能是异质的——浅层做细节、深层做语义，放大模型时必须协调两边的比例，没有简单的放大公式。

DiT 的同质结构则只有一个旋钮：堆更多 Transformer Block。

每一档模型的放大方式完全一致，FID（图像质量分数）随参数量的提升高度可预测，这正是 NLP 领域 Scaling Law 的迁移。对于大厂来说，这意味着”堆算力就能堆质量”——战略价值巨大。

去噪任务其实不需要跳跃连接

前面说 U-Net 的跳跃连接是精髓，但在去噪任务里，它的必要性其实存疑。

图像分割需要跳跃连接，是因为输出（标签图）和输入（原图）的信息来源完全不同，必须在解码时补回编码器丢失的细节。

扩散模型的去噪则不同：输入本身就是一张（有噪声的）图像，模型预测的噪声图和输入在结构上高度相关。Transformer 的全局 Attention 足以在没有跳跃连接的情况下维持这种全局一致性，实验结果也证明 DiT 的生成质量全面超越同等算力的 U-Net 方案。

一句话总结两者的差异

U-Net（卷积） DiT（Transformer）
全局感受野 间接，靠堆层数 直接，一层 Attention
架构形状 U 形，不对称 等深等宽，同质
Scaling 不规律，难预测 规律，可预测
放大方式 需协调编解码两侧 直接堆 Block
设计起源 为分割任务定制 为序列建模定制

U-Net 在图像生成领域的退场，并非因为它”不好用”，而是因为扩散模型的核心需求（全局建模、大规模 Scaling）恰好是 Transformer 的强项，而不是卷积的强项。这是架构适配度的问题，不是优劣问题。

10. 总结

核心概念 一句话
通道数 从多少个角度观察图像，越深越多
下采样 压缩空间换取更大视野，代价是丢失细节
上采样 还原空间尺寸，配合跳跃连接补回细节
跳跃连接 把编码器的”快照”直接送给解码器，防止细节永久丢失
拼接 在通道维度合并两个特征图，尺寸不变，通道数相加
U-Net 精髓 编码器理解语义 + 跳跃连接保留细节 + 解码器还原尺寸
为何用于扩散 去噪任务和分割任务形式相同（输入=输出尺寸），加条件注入即可

U-Net 是一个非常优雅的设计——它用最直觉的方式解决了”理解”与”定位”之间的矛盾，并因这种通用性被跨领域借用，在 AI 生成图像的早期时代发挥了核心作用。它被 DiT 取代，不是因为设计失败，而是因为扩散模型的规模化需求，与 Transformer 的特长恰好完美匹配。理解 U-Net 的设计与退场，是理解整个图像生成技术演进路径的必要基础。
